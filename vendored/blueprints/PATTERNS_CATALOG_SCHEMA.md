# `patterns-catalog.yaml` schema

Top-level machine-readable index of every pattern and workflow in this repo. Aggregated from per-pattern `metadata.json` plus tier-file presence on disk plus the composition matrix. Auto-generated by [`meta/validate-metadata.js --emit`](./meta/validate-metadata.js); never edit by hand.

The primary consumer is **agent-deployments' CI**, which fetches this file at generation time and embeds the relevant fields into its own `catalog.yaml`. Downstream tools (e.g. `agent-scaffold`) read the deployments catalog, not this one — so the contract surface that needs to stay stable is narrow.

## Regenerating

```bash
node meta/validate-metadata.js --emit patterns-catalog.yaml
```

The generator is deterministic: same source files → byte-identical output. The drift workflow (`.github/workflows/catalog-drift.yml`) regenerates on every PR and fails if the committed file diverges.

## Top-level shape

```yaml
schema_version: 1
generator_version: 1.0.0

patterns:       # agent patterns — flow controlled by the LLM
  - id: react
    name: ReAct
    category: agent
    complexity: Intermediate
    description: "..."
    dir: patterns/react
    tier_files:
      overview: patterns/react/overview.md
      design: patterns/react/design.md
      implementation: patterns/react/implementation.md
      evolution: patterns/react/evolution.md
      observability: patterns/react/observability.md
      cost-and-latency: patterns/react/cost-and-latency.md
    evolvesFrom: [prompt-chaining]
    composableWith: [memory, reflection, rag]
    requires: [tools]
    tags: [reasoning, tool_use, loop, adaptive, open-ended]
    costTier: medium
    latencyTier: variable
    extras:
      prompts: patterns/react/prompts/
      schemas: patterns/react/schemas/
      code: patterns/react/code/

workflows:      # workflow patterns — flow controlled by code
  - id: prompt-chaining
    name: Prompt Chaining
    category: workflow
    complexity: Beginner
    description: "..."
    dir: workflows/prompt-chaining
    tier_files:
      overview: workflows/prompt-chaining/overview.md
      design: workflows/prompt-chaining/design.md
      implementation: workflows/prompt-chaining/implementation.md
      observability: workflows/prompt-chaining/observability.md
      cost-and-latency: workflows/prompt-chaining/cost-and-latency.md
    evolvesInto: [react, tool_use, memory]
    composableWith: [parallel-calls, evaluator-optimizer]
    tags: [sequential, deterministic, predictable, low-latency]
    costTier: low
    latencyTier: medium

compositions:   # cross-pattern relationships parsed from composition/combination-matrix.md
  - {a: rag, b: react, kind: natural, rationale: "ReAct uses retrieval as one of its tools"}
  - {a: routing, b: reflection, kind: complex, rationale: "Reflect on routing"}
```

## Field semantics

### Top-level

| Field | Type | Notes |
|---|---|---|
| `schema_version` | int | Bumped on breaking field/semantic changes. Consumers refuse versions above their declared max. |
| `generator_version` | string | Bumped manually when the generator's output format changes in a non-breaking way. Diagnostic only. |
| `patterns` | array | Entries where `category == "agent"`. Sorted by `id`. |
| `workflows` | array | Entries where `category == "workflow"`. Sorted by `id`. |
| `compositions` | array | Cross-pattern relationship edges from the matrix. Sorted by `(a, b)`. |

The catalog deliberately **omits** `source_commit_sha` and `generated_at`. Those would create non-determinism (commit SHA tail-eating; timestamp churn) and break the drift CI's byte-diff. Consumers that need the upstream commit query it via the GitHub API at fetch time.

### Pattern / workflow entry

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | string | yes | Matches the directory name; the canonical identifier. |
| `name` | string | yes | Display name (used as a bridge to the composition matrix). |
| `category` | enum | yes | `"agent"` or `"workflow"`. |
| `complexity` | enum | yes | `"Beginner"`, `"Intermediate"`, or `"Advanced"`. |
| `description` | string | yes | One-sentence summary. |
| `dir` | string | yes | Repo-root-relative path to the pattern directory. |
| `tier_files` | map | yes | Map of tier name → relative path. Only includes tiers whose `.md` file exists on disk. |
| `evolvesFrom` | string[] | no | IDs of patterns this evolved from. Agent patterns only. |
| `evolvesInto` | string[] | no | IDs of agent patterns this workflow evolves into. Workflow patterns only. |
| `composableWith` | string[] | no | IDs of patterns that compose naturally with this one. |
| `requires` | string[] | no | Domain prerequisites (e.g. `"tools"`, `"vector-store"`). |
| `tags` | string[] | no | Searchable keywords. |
| `costTier` | string | no | `"low"`, `"low-medium"`, `"medium"`, `"high"`, `"variable"`. |
| `latencyTier` | string | no | Same vocabulary as `costTier`. |
| `extras` | map | no | Best-effort detection of optional subdirs (`prompts/`, `schemas/`, `code/`, `examples/`). Only present when at least one is detected. |

### Composition edge

| Field | Type | Required | Notes |
|---|---|---|---|
| `a` | string | yes | First pattern id, lexicographically smaller of the pair. |
| `b` | string | yes | Second pattern id, lexicographically larger of the pair. |
| `kind` | enum | yes | `"natural"`, `"useful"`, `"complex"`, `"redundant"`, or `"anti"`. |
| `rationale` | string | no | One-line explanation, lifted from the matrix cell. Omitted when the cell has only `**Kind**` with no em-dash explanation. |

Edges are undirected: each unordered pair appears at most once, with `a < b` lexicographically.

## Schema version policy

- **v1** (this release) — initial.
- Additive fields don't bump `schema_version`. Consumers must use forward-compatible parsing (ignore unknown fields).
- Removing a field, changing a field's type, or changing semantics bumps `schema_version`. Coordinate version bumps with the `agent-deployments` maintainer so deployments-CI updates in lockstep.

## Consumer contract

The primary consumer is `agent-deployments/scripts/generate_catalog.py`:

1. Fetches `https://raw.githubusercontent.com/jagguvarma15/agent-blueprints/main/patterns-catalog.yaml`.
2. Queries GitHub Commits API for the HEAD SHA on `main`; stamps as `blueprints.upstream_sha` in the deployments catalog.
3. Embeds `patterns[]`, `workflows[]`, and `compositions[]` into deployments' own `catalog.yaml`.

Downstream consumers (`agent-scaffold` etc.) read the deployments catalog, not this file directly.

## Drift CI

`.github/workflows/catalog-drift.yml` runs on every PR and push to `main`:

```yaml
- run: node meta/validate-metadata.js --emit /tmp/patterns-catalog.yaml
- run: |
    if ! diff -u patterns-catalog.yaml /tmp/patterns-catalog.yaml; then
      echo "::error::patterns-catalog.yaml is stale. Run 'node meta/validate-metadata.js --emit patterns-catalog.yaml' and commit."
      exit 1
    fi
```

The drift check is a hard gate: the only way to update the catalog is to regenerate it and commit the result.
